ЗАМЕТКИ ПО ПРОГНОЗИРОВАНИЮ НАГРУЗКИ НА СЕРВЕР

Прогнозирование нагрузки на сервер — задача предсказания будущих значений метрик (CPU, RAM, сетевой трафик, количество запросов) на основе исторических данных. Цель: обеспечить стабильную работу, автоматическое масштабирование, предотвращение перегрузок.

КЛЮЧЕВЫЕ ПОНЯТИЯ:
- Временной ряд — последовательность измерений нагрузки, упорядоченная по времени.
- Тренд — долгосрочное направление изменения нагрузки (рост, падение).
- Сезонность — регулярные повторяющиеся паттерны (например, увеличение трафика днём, снижение ночью; пики в конце месяца).
- Шум — случайные колебания, не связанные с закономерностями.
- Горизонт прогноза — период времени, на который делается прогноз (например, 1 час, 24 часа).
- Лаги — значения нагрузки в предыдущие моменты времени, используемые как признаки.

ОСНОВНЫЕ ЭТАПЫ ПОСТРОЕНИЯ ПРОГНОЗА:
1) Сбор и подготовка данных
   - Собираем метрики с серверов (CPU, memory, disk I/O, network, количество запросов в секунду).
   - Обрабатываем пропуски (интерполяция, заполнение средним).
   - Удаляем выбросы (например, вызванные разовыми сбоями).
   - Приводим к единой временной сетке (ресамплинг).

2) Анализ и визуализация
   - Строим графики временных рядов.
   - Оцениваем стационарность (можно использовать тест Дики-Фуллера).
   - Выявляем сезонность, тренд, цикличность (декомпозиция ряда).
   - Изучаем корреляцию с внешними факторами (время суток, день недели, праздники, релизы).

3) Инжиниринг признаков
   - Создаём признаки на основе календаря: час, день недели, месяц, признак выходного/праздничного дня.
   - Добавляем лаговые значения (значения нагрузки в предыдущие моменты).
   - Скользящие статистики (средние, максимумы за окно).
   - Внешние данные: если есть, например, данные о планируемых нагрузках, рекламных акциях.

4) Выбор модели
   - Классические статистические методы: ARIMA, SARIMA, Exponential Smoothing (Holt-Winters).
   - Методы машинного обучения: линейная регрессия, случайный лес, градиентный бустинг (XGBoost, LightGBM, CatBoost).
   - Глубокое обучение: LSTM, Transformer-модели (например, Informer, Autoformer).
   - Ансамбли и комбинации моделей.

5) Обучение и валидация
   - Разделяем данные на обучающую и тестовую выборки с учётом времени (time series split).
   - Обучаем модель, подбираем гиперпараметры.
   - Оцениваем качество метриками: MAE, RMSE, MAPE, sMAPE, MASE.

6) Интерпретация и использование
   - Анализируем важность признаков (для моделей типа бустинга).
   - Строим доверительные интервалы прогноза.
   - Интегрируем прогноз в систему мониторинга и автоскейлинга.

ЧАСТО ИСПОЛЬЗУЕМЫЕ БИБЛИОТЕКИ (PYTHON):
- pandas, numpy — обработка данных.
- matplotlib, seaborn, plotly — визуализация.
- statsmodels — статистические методы (ARIMA, декомпозиция).
- scikit-learn — модели машинного обучения, метрики.
- xgboost, lightgbm, catboost — градиентный бустинг.
- prophet (Facebook) — модель для временных рядов с сезонностью.
- pytorch, tensorflow — нейросети (LSTM и др.).
- sktime, darts — специализированные библиотеки для временных рядов.

ТИПИЧНЫЕ ПРОБЛЕМЫ И ПУТИ РЕШЕНИЯ:
- Нестационарность ряда → дифференцирование, преобразования (логарифм), использование моделей, устойчивых к тренду.
- Сложная множественная сезонность → добавление соответствующих признаков, использование моделей с поддержкой нескольких сезонов (например, Prophet).
- Редкие пиковые нагрузки → включение информации о событиях, использование квантильных моделей для оценки рисков.
- Долгосрочные прогнозы → комбинация моделей для разных горизонтов, иерархический подход.
- Запаздывание прогноза относительно реальности → уменьшение горизонта, использование более коротких лагов.

ПРИМЕР MVP (минимально жизнеспособного продукта):
1. Собираем исторические данные по CPU за последние 30 дней с часовым разрешением.
2. Строим простую модель (например, скользящее среднее с учётом сезонности суток).
3. Делаем прогноз на следующий час.
4. Визуализируем на дашборде.
5. При превышении порога — отправляем алерт.
